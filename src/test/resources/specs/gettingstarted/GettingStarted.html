For testing validation, parametrized (aka where) cases may be used:

{"name": "widget1"} { "error": "quantity is required" } {"quantity": "10"} { "error": "name is required" } {"name": "{{invalid}}", "quantity": "10"} { "error": "{{error}}" } '' , blank value not allowed more_than_10, Value 'more_than_10' can't be stored to database column because exceeds length app.Widgets.name.columnType.colLength

Or more tricky and compact equivalent of previous example (request/response body templates are hidden in files):

ignored , invalid-no-name.json , name is required ignored , invalid-no-quantity.json, quantity is required '' , create-req.json , blank value not allowed more_than_10, create-req.json , Value 'more_than_10' can't be stored to database column because exceeds length app.Widgets.name.columnType.colLength
