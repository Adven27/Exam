DELETE

Specifying use cases of a DELETE-endpoint

Overview

e:delete
<e:delete url="/some/url" contentType="application/json">
    <!-- One or many cases -->
    <e:case desc="use-case 1">
        <!--  Expected response -->
        <e:expected statusCode="200">
            { "response": "body" }
        </e:expected>
        <!-- Optional block for case specific checks -->
        <e:check>
            <span c:assertTrue="true">Block for additional checks</span>
        </e:check>
    </e:case>
    <e:case desc="use-case 2">
        <e:expected from="/ws/response.json" statusCode="200"/>
    </e:case>
</e:get>

Usage

1. Configure the connection to the Web Service via WsPlugin.

2. Use e:delete tag in specification:

src/test/resources/specs/Specs.html

<e:delete url="/some/url" contentType="application/json">
    <!-- One or many cases -->
    <e:case desc="use-case 1">
        <!--  Expected response -->
        <e:expected statusCode="200">
            { "response": "body" }
        </e:expected>
        <!-- Optional block for case specific checks -->
        <e:check>
            <span c:assertTrue="true">Block for additional checks</span>
        </e:check>
    </e:case>
    <e:case desc="use-case 2">
        <e:expected from="/ws/response.json" statusCode="200"/>
    </e:case>
</e:get>

Examples

Query string

To specify case specific query string case.urlParams attribute may be used.

urlParams

urlParams
The following markup:
<e:delete url="/mirror/request">
  <e:case desc="Request with single param" urlParams="param1=1">
    <e:expected> { "DELETE": "/mirror/request?param1=1" } </e:expected>
  </e:case>
  <e:case desc="Request with several params" urlParams="param1=1&amp;param2=2">
    <e:expected> { "DELETE": "/mirror/request?param1=1&amp;param2=2" } </e:expected>
  </e:case>
</e:delete>
will be rendered as:
DELETE /mirror/request
Content-Type application/json
